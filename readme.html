Scheduled Events — for Zen Cart v1.5.1 through v1.5.5e

Version 2.0.0

Current Support Thread at Zen Cart Forums: Scheduled Events Add-On [Support Thread]

This plugin represents a restructuring of the Scheduled Events, v1.7.0 for Zen Cart v1.3.9-v1.5.5, enabling the plugin to support current Zen Cart and PHP versions and including the creation, deletion, and insertion of Scheduled Events through the Admin Menu versus the previous, separate, outside method.

Purpose

This plugin removes the need for "offsite" creation, deletion, and correction of Scheduled Events with an admin-level tool Localization->Scheduled Events, which are displayed on the Scheduled Events home page.

The Scheduled Events have a Start Date and End Date (which can be set to "--none--"), which determine whether or not the Scheduled Event is displayed to your customers. Events are displayed if the Event's status is Enabled, the Start Date is on or after the current date and the End Date is either "--none--" or on or before the current date. That way, you can enter an Event today that isn't displayed to your customers until sometime next week.

Admin Interfaces

Once you have copied the admin-level files to your cart's file system, you will see two new menu items:

  1. Configuration->Scheduled Events Settings, where you control the plugin's settings.
  2. Localization->Scheduled Events Manager, where you write and manage your store's Scheduled Events. If you have an HTML editor installed in your admin, you can use that editor to edit/create your Scheduled Events. If you have multiple languages in your store, you can write language-specific Events, also.

Customer Interfaces

On your store-side, once you have installed and enabled the plugin there are two new elements to be shown to your customers:

  1. The Scheduled Events selection in the Information Sidebox. Provides a link to the most-recent Scheduled Events.
  2. The Scheduled Events page. Displays the full list of active Scheduled Events.

Installation

This plugin has no core-file overwrite, but one template-override file; you should always backup your cart's database and files prior to making any changes.

  1. Unzip the plugin's package file, which you've most likely already done if you're reading this!
  2. Rename the /YOUR_ADMIN directory to match your Zen Cart's secret admin-folder name.
  3. Rename the one template-override directory (/includes/modules/YOUR_TEMPLATE) to match your cart's active template folder.
  4. If you've made previous changes to /includes/modules/sideboxes/YOUR_TEMPLATE/information.php, you'll need to merge those changes with this plugin's updates (one marked section).
  5. Log into your Zen Cart's admin-console.
  6. Copy only the plugin's admin-level files to your store's file system.
    1. /YOUR_ADMIN/news_box_manager.php
    2. /YOUR_ADMIN/includes/auto_loaders/config.news_box_manager.php
    3. /YOUR_ADMIN/includes/extra_datafiles/news_box_manager_file_database_names.php
    4. /YOUR_ADMIN/includes/extra_datafiles/news_box_manager_sanitization.php (Added in v2.1.1)
    5. /YOUR_ADMIN/includes/init_includes/init_news_box_manager_admin.php
    6. /YOUR_ADMIN/includes/languages/english/news_box_manager.php
    7. /YOUR_ADMIN/includes/languages/english/extra_definitions/news_box_manager_menu_name.php
    8. /YOUR_ADMIN/includes/languages/german/news_box_manager.php
    9. /YOUR_ADMIN/includes/languages/german/extra_definitions/news_box_manager_menu_name.php
  7. Click the "Admin Home" link in the admin header; that will cause this plugin's initialization script to run, creating the necessary database tables and configuration items.
  8. Copy the plugin's store-side files to your store's file system.
    1. /includes/extra_datafiles/events_extra_pages_filenames.php
    2. /includes/extra_datafiles/events_extra_tables_filenames.php
    3. /includes/extra_datafiles/events_filenames.php
    4. /includes/languages/english/events.php
    5. /includes/modules/sideboxes/YOUR_TEMPLATE/information.php
    6. /includes/templates/YOUR_TEMPLATE/templates/tpl_events_default.php
  9. Go to your admin's Configuration->Scheduled Events Settings to view and change any settings, then to to Localization->Scheduled Events Manager to start writing!

Un-Installing

To un-install this plugin, first remove all the new files introduced by this plugin and restore the template-override file. Then open the plugin's /uninstall_scheduled_events.sql, copy and paste the file's contents into your admin's Tools->Install SQL Patches and click "Send". That will remove all configuration settings and plugin-specific tables from your database.
